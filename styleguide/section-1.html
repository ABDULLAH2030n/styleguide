1Style Rules #permalink

Here are detailed all the style rules for the web application.

1.1Depth #permalink

Getting a deeper look into the styles of shadows and positioning.

Keep the number of stacked layers to a minimum and the total number of layer below 3 (including the background).

Layer 1
Layer 2

</div>

1.2Lists #permalink

Options can only effect the list item it is a part of. Options can have a tooltip and an icon and they are displayed only if the item is selected. It is the "leave-behinds" concept : http://www.google.com/design/spec/components/list-controls.html#list-controls-types-of-list-controls

Note : It's also good to provide totals at each column bottom in a grid when it's relevant.

1.6Buttons #permalink

Avoid non specific actions as a label of an action. For example, "OK" and "GO" are not specific but "SAVE USER" and "GO ON DASHBOARD" are.

1.6.1Default buttons #permalink

The default button is 'printed' on the material and only show its area when hovered, it uses capital letters to differentiate it from a link. The Secondary buttons area is always visible and should mainly be used for functions.

Example 
<button class="wfm-btn wfm-btn-invis-default">DEFAULT</button>
<button class="wfm-btn wfm-btn-invis-primary">PRIMARY</button>
<button class="wfm-btn wfm-btn-default material-depth-1">Primary</button>
<button class="wfm-btn wfm-btn-primary material-depth-1">Primary</button>
<button class="wfm-btn wfm-btn-invis-disabled">DISABLED</button>
<button class="wfm-btn wfm-fab">+</button>

1.8Tables #permalink

If a table take time to be loaded, a loading bar should be displayed during this time. A table must be displayed in alphabetical order, except for specific use case. In general, providing ways to filter and sort on tables, lists, grid is good.

1.10Alerts #permalink

There are several different error markers. The default one is used to provide information to the user. The warning notice should be used to alert the user that something have to be done before they can continue. The error notice should only be triggered when something goes wrong in the background.

Example
Did you know that this can be expanded to show details?
Well you do now
Agen Lisas contract could not be fulfilled.
Something triggered this error notice.
<section class="expandable">
	<input id="ac-1" name="accordion-1" type="checkbox" checked="false">
	<label for="ac-1" class="main-row notice-default">This is a default notice. <span>></span></label>
	<article class="ac-small">
		<div class="main-row notice-detail">Did you know that this can be expanded to show details?</div>
		<div class="main-row notice-detail">Well you do now</div>
	</article>
</section>
<section class="expandable">
	<input id="ac-2" name="accordion-2" type="checkbox" checked="false">
	<label for="ac-2" class="main-row notice-warning">Hey, have a look at this warning notice.<span>></span></label>
	<article class="ac-small">
		<div class="main-row notice-detail">Agen Lisas contract could not be fulfilled.</div>
	</article>
</section>
<div class="main-row notice-error">Something triggered this error notice.</div>
